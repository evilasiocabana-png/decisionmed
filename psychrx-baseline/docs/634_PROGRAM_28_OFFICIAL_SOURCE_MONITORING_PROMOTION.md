# 634 - Program 28 Official-Source Monitoring Promotion

## Objective

Resolve the four monitoring publication decisions identified by Program 27 and
reconcile the remaining pharmacological backlog against its actual scientific
state.

## Missions

1. approve an ADR before changing runtime behavior;
2. create section-level official source anchors;
3. publish only non-prescriptive `MonitoringPlan` knowledge;
4. require patient-state trigger matching;
5. audit interaction, disease-use and Motor 2 rows without mass promotion;
6. run full regression, source and architecture audits.

## Files Allowed

- one new ADR;
- monitoring knowledge tables;
- monitoring application orchestration;
- pharmacological coverage audit;
- proportional tests and certification documents.

## Files Prohibited

- founder documents 000-006;
- autonomous diagnosis or prescription engines;
- dose or taper calculators;
- unreviewed mass changes to pharmacological evidence status.

## Acceptance Criteria

- four monitoring rules pass scientific, editorial and runtime gates;
- rule output is source cited and context triggered;
- a failed gate prevents activation;
- every quoted backlog number is reproducible;
- no pending pharmacological row is presented as validated;
- full regression remains green.

## Declaration Final

Program 28 authorizes a small, reversible monitoring increment. It does not
authorize scientific promotion by inference, source discovery alone or row
count manipulation.
