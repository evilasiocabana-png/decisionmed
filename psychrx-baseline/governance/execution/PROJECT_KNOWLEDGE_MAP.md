# Project Knowledge Map - PsychRx

## Funcao

Este documento mapeia os dominios de conhecimento que o PsychRx precisa organizar ao longo do desenvolvimento.

## Mapa Principal

```text
Clinical Foundation
-> Psychopharmacology
-> Evidence
-> Safety
-> Clinical Reasoning
-> Monitoring
-> Longitudinal Outcomes
-> Clinical Experience
-> Validation
```

## Dominios

| Dominio | Conteudo |
| --- | --- |
| Clinical Foundation | Constituicao Clinica, paciente, sintomas, hipoteses, objetivos e limites. |
| Psychopharmacology | Agentes, classes, efeitos, interacoes, contraindicacoes e monitorizacao. |
| Evidence | Diretrizes, revisoes, meta-analises, ensaios, consensos e regulacoes. |
| Safety | Riscos, alertas, bloqueios, encaminhamento e monitorizacao obrigatoria. |
| Clinical Reasoning | Snapshot, perguntas, diagnostico, objetivos, restricoes, estrategias e explicacao. |
| Monitoring | Resposta, eventos adversos, funcionalidade, recaida, recorrencia, remissao e estabilizacao. |
| Clinical Experience | Consulta, cards, anamnese guiada, perguntas ao vivo e modo paciente. |
| Validation | Testes, simulacoes, auditoria, revisao cientifica e maturidade. |

## Relacao com Knowledge Graph

O Knowledge Graph organiza relacoes entre elementos clinicos.

O Project Knowledge Map organiza areas de conhecimento do projeto.

## Declaracao Final

O Project Knowledge Map ajuda a impedir que conhecimento clinico, software e experiencia de usuario sejam tratados como a mesma coisa.
