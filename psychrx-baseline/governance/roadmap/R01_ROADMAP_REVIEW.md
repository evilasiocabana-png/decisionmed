# R01 Roadmap Review

## Reference

REF: CQ-R01-002

## Mission

R01-002 - ROADMAP_REFACTORING_REVIEW.

## Objective

Review the compressed roadmap created in R01-001 and determine whether the 200 mission model preserves the full scope of PsychRx.

## Review Scope

The review compared:

```text
Original Roadmap
-> Programs 00-26
-> Program X01
-> Track A A01-A15
-> A02.5 / A04.0 corpus programs
```

against:

```text
Refactored Roadmap
-> Programs 00-13
-> 176 mission slots
-> parameterized pipelines
```

## Review Questions

### 1. Was any functionality lost?

No critical functionality was lost.

Most original capabilities are represented in the compressed model as either:

- core programs;
- reusable pipelines;
- explicit gates;
- historical artifacts preserved outside the compressed execution count.

### 2. Did any gate disappear?

No mandatory gate disappeared.

However, the compressed model must explicitly preserve:

- Corpus Gate;
- Metadata Gate;
- Source Validation Gate;
- Extraction Protocol Gate;
- Drug Portfolio Gate;
- Scientific Extraction Gate;
- Normalization Gate;
- Scientific Review Gate;
- Editorial Review Gate;
- Publication Gate;
- Runtime Eligibility Gate.

### 3. Did any pipeline become incomplete?

No core pipeline is missing.

The pipelines remain conceptually complete, but R01-003 must define execution templates for each pipeline before the model becomes operational.

### 4. Did any phase lose representation?

No major phase lost representation.

Detailed class-specific phases are compressed into parameterized executions.

### 5. Did any program disappear without replacement?

No.

Class-specific programs are mapped to:

- `Scientific Corpus Pipeline[DrugClass]`
- `Drug Class Population Pipeline[DrugClass]`
- `Evidence Integration Pipeline[DrugClass]`
- `QA and Publication Pipeline[DrugClass]`

### 6. Are all pharmacological classes still supported?

Yes.

The model supports:

- SSRIs;
- SNRIs;
- NDRIs;
- NaSSAs;
- TCAs;
- MAOIs;
- SGAs;
- FGAs;
- Mood Stabilizers;
- Anxiolytics;
- ADHD medications;
- Cognitive enhancers.

### 7. Is the model still scalable?

Yes.

The compressed model is more scalable because it parameterizes repeated execution instead of duplicating programs.

## CTO Review Result

The roadmap compression is acceptable as a governance direction.

It should not immediately replace the current execution queue until R01-003 creates the migration execution plan.

## Decision

Approve conditionally.

## Conditions

- R01-003 must define how active work transitions from `A04.0-005` to the pipeline model.
- Historical artifacts must remain untouched.
- Pipeline templates must preserve gates.
- Runtime eligibility remains blocked.
- No scientific content may be generated by compression alone.

## Final Declaration

The compressed roadmap preserves PsychRx capabilities at the governance level, but it requires an execution migration plan before becoming the active operating model.
